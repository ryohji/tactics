# 成果物テンプレート

`docs/` 配下の共有ドキュメントの雛形。最初から完璧を狙わず、分かったことから埋め、フェーズが進むたびに更新する。
空欄やプレースホルダを残してよい — 「まだ決まっていない」ことが可視化されているほうが、決め忘れを防げる。

---

## docs/VISION.md

```markdown
# ビジョン

## 体験の核
プレイヤーに何を感じてほしいか(一文で)。

## ターゲット
誰のためのゲームか。

## なぜ面白いか / 差別化
既存のタクティクスゲームと何が違い、なぜそれが良いか。

## 北極星(North Star)
迷ったときに立ち返る一つの判断基準。
```

---

## docs/REQUIREMENTS.md

```markdown
# 要求定義

## MVP スコープ
最初に「触れる形」で見せるもの。

## 機能リスト
- [ ] 機能A — 概要 / 優先度
- [ ] 機能B — 概要 / 優先度

## ゲームルール(現時点の決定)
盤面、ターン進行、ユニット、行動、勝敗条件 など分かっている範囲。

## 制約
技術・期間・その他の制約。

## 非目標(やらないこと)
スコープ膨張を防ぐため、あえてやらないことを明記する。
```

## docs/DESIGN.md

```markdown
# 設計

## アーキテクチャ概要
Web 版前提の全体構成(レンダリング / 状態管理 / 入力 / ロジック分離)。

## ゲームモデル
- 盤面: グリッド種別(方眼/ヘックス)、座標系
- ユニット: 属性、状態
- ターン / 行動解決: 順序、ルールの処理フロー

## 技術選定
| 項目 | 採用 | 理由 | 代替案 |
|---|---|---|---|

## 作業分解(今イテレーション)
- 単位1: 目的 / 対象 / 完了条件
- 単位2: ...
```
