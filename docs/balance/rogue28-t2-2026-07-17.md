# バランスシミュレータ(rogue-28 T2 時点・2026-07-17)

`npm run balance -- --seeds 50 --out docs/balance/rogue28-t2-2026-07-17.jsonl` の結果。
シード 1..50 を greedy/cautious それぞれ1回ずつ(計100ラン)、`setTimeScaleForTest(0)` で
演出待ちを飛ばしてヘッドレス実行。実行時間 155.5秒。

rogue-28 T2 の変更点(ボットに影響するもの):
- throwItem アクション追加(pack のアイテムを敵へ投擲・武具は落下・水薬は消滅)。
  ただしボットの方策(greedy/cautious)は投擲を使わないため、この実装は
  本計測には現れない(UI 未実装のため)。
- 保持上限 10 枠により拾得ロジックが変更(満杯時は stackable な新規は拾えず)。
  但し初期パック容量と深度内の lootTable に大きな変化がないため、実測への影響は軽微。

## 一言

T2 の実装(投擲アクション+保持上限)は UI がないため、ボットの行動にまったく影響せず、
既存の golden と balance は不変。

T3 でUI を付けたとき、人間プレイヤーが投擲を活用すればゲームバランスが変わる可能性
あり(例: 射程制限外の敵に水薬を投げて削る、など)。その時点での再測定推奨。
