# 要求定義

土台となる表示・移動ルーチンは `fcc_hex_3d_strategy_spec.md` に確定済み(座標系・FCC・菱形十二面体・[111]鉛直・カメラ/LoS/脅威場)。本書はその上で**最初に検証する一点**を定める。

## MVP スコープ
**3D地形 × 3Dヘックスの合成プロトタイプ。**
真の3Dボリューム地形を1つ構成し、FCCアリーナと重ね合わせ、地形に重なるセルを進入禁止+遮蔽として分類する。ユニット1体を置いて到達セル/遮蔽を表示し、カメラで動かして**表示・操作に違和感がないことを確認できる**状態をゴールとする。

## 機能リスト(優先度: ◎必須 / ○あれば)
- [ ] ◎ FCCアリーナ生成(仕様5章の六角柱。`Lmin/Lmax/Hmax`)
- [ ] ◎ `worldPos` 変換と基本レンダリング(空きセルの格子点マーカー=淡い小球)
- [ ] ◎ 真の3Dボリューム地形を1つ構成(オーバーハング・洞窟を含みうる形状。実験用に1形状でよい)
- [ ] ◎ 地形×セルの重なり判定: **セル中心から距離 d 以内に地形があれば進入禁止+遮蔽** → `occluderSet`
- [ ] ◎ 地形の描画(ボリュームと、菱形十二面体の塗りなど胞構造と整合する見せ方)
- [ ] ◎ `reachable`(地形・占有を除外した到達セル)の算出と強調表示
- [ ] ◎ TPカメラ(自己中心旋回)+ クリックによる移動(レイキャスト→到達セルなら遷移→再評価)
- [ ] ◎ `d` をはじめパラメータの実時間調整(スライダ/トグル。違和感の当たりを取るため)
- [ ] ○ 遮蔽フォグ(`lineBlocked` による視界の塗り分け)
- [ ] ○ 脅威圏表示(将来のゲーム層に向けた素振り。敵を仮置きできれば)
- [ ] ○ FP視点

## ゲームルール(現時点の決定)
- 1手 = 1ステップ移動(`OFFSETS` の12近傍)のみ。
- ユニット種別・行動経済・勝敗条件は**範囲外**(基盤の手触り検証に集中する)。

## 制約
- Web 版(Three.js / React Three Fiber 系)で実装する。
- 自分/少数の実験向け。性能は中規模アリーナで快適なら十分(大規模最適化は後回し)。
- 仕様書の座標系・幾何定数(`OFFSETS` / `BASIS` / `RD_VERTICES` 等)に厳密準拠する。

## it-4 更新(2026-06-29: 廃墟大聖堂 / 壮大スケール / 飛行)
- 地形を **廃墟化した大聖堂**(壁が一部崩れ内部露出)へ。人間サイズ ≈ 1セルの**壮大スケール**。
- ユニットは飛行できる翼人・魔女。**全ユニット飛行**=室内外の3D空間すべてが移動可能、地形=実体occluderのみ(移動ロジックは現 `reachable` のままで成立)。
- 壮大スケール対応の描画: **空きセル全描画を廃止**し、地形occluder+到達セル(クリック対象)+脅威+ユニットを描く。raycast は click 時のみ、d 変更はスロットル。
- 性能最適化はこの段で**部分的に解禁**(壮大スケールの実用性のため)。

## 非目標(やらないこと)
- 勝敗条件・複数ユニットのAI・マルチプレイ(ユニットは仮置きのまま)。
- 地形の手続き生成の一般化(実験用の1形状で足りる)。
- Unity/Godot 等エンジンへの移植(展開段階で別途検討)。
- ※「インスタンシング等の性能最適化は非目標」は it-4 で見直し(壮大スケールのため部分解禁)。
