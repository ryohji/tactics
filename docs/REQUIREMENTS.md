# 要求定義

現行プロダクトは **FCC Rogue — 蟻巣迷宮**(蟻の巣状 3D ダンジョンを探索する
ターン制ローグライク)。土台の座標系・幾何(FCC 格子・菱形十二面体・[111]鉛直)は
`fcc_hex_3d_strategy_spec.md` に確定済み。前身の3Dタクティクス試作(it-1〜6)の
要求は圧縮した(経緯は DECISIONS 2026-06-27〜07-03、コードは `tactics` ブランチ)。

## 中核要求(rogue-1、2026-07-04 — 達成済み)
- ダンジョンのオンデマンド生成(広間+通路。進行に応じて必要なだけ。下向きバイアス)
- 一定距離で気づく敵(縄張り・階層下限)/ 武器・防具・消耗品(回復/投擲)のトレジャー
- 死亡→スコア(最深到達・討伐)→再挑戦。勝利条件なしのエンドレス潜行

## その後の確定(rogue-2〜18 — 達成済み)
- 探索支援: バブル(名前+歩数)・ファストトラベル(タップ/クリック/ESC で中断)・
  マップモード・TAB ターゲット巡回
- 明かり3段階のトレードオフ(視界/回復 ↔ 敵の気づき)/ アイテム品質と合成 /
  罠5種+状態異常 / 砲塔・囮
- シードによるプレイ再現(生成・戦闘乱数の完全決定論)/ 毎ターン自動保存(localStorage)
- スマートフォン対応(2段階タップ・レスポンシブ HUD)
- 見た目と音: カットアウェイ描画・glTF モデル・WebAudio 合成の SE/深度連動 BGM

## その後の確定・第2期(rogue-19〜25 — 達成済み)
- 層による探索打ち切り(深度8ごとの関門+崩落。メモリ・セーブ容量を1層分で頭打ち)
- バランスシミュレータ(`npm run balance`。方策×シードの統計と変更前後の A/B 差分)
- ローカルスコアボード(ラン履歴・自己ベスト・本日の迷宮)+行動ログの記録
- 障壁・状態異常(毒/混乱)・盾と両手武器・敵ロスター拡張(17種+門番3種・遠隔・
  気配感知・深度係数)—「対抗手段が先、脅威が後」の配置原則
- スキル体系(マスタリー7系統×スロット×関門ドラフト・全25ノード。死んでも残る
  永続メタと、ラン内でリセットされる力の二層構造)
- コレクション(図鑑・実績8種・遺物「巣の琥珀」・脱出=生還と展示棚)

## 次期要求(rogue-26)
- 共有スコアボード: ゲーム側は送信のみ、Cloudflare Worker(無料枠)が記録と
  top100 保持。バージョン記録・スキル構成の併記・関門スナップショット送信。
  発展形はシード+行動ログのリプレイ検証(詳細は ROADMAP バックログ)

## 制約
- Web 版(Three.js / React Three Fiber)。自分/少数の実験向け。性能は中規模で快適なら十分
- **決定論**: ゲーム内乱数はすべてランのシードから導出する(Math.random 禁止)。
  プレイ再現・ゴールデンテスト・将来のリプレイ検証の前提
- 外部アセットは再配布可能ライセンス(CC0/CC-BY)の 3D モデルのみ
  (rogue-13 で解禁。出典を `public/models/` に同梱)。音は WebAudio 合成、
  テクスチャは canvas 生成を維持
- 幾何は仕様書の座標系・定数(`OFFSETS` / `BASIS` / `RD_VERTICES` 等)に厳密準拠

## 非目標(やらないこと)
- マルチプレイ(共有スコアボードは非同期の競争であり対象外)・キャンペーン
- Unity/Godot 等エンジンへの移植(展開段階で別途検討)
- 履歴: 「セーブデータなし」は rogue-8 で解除(自動保存)/「外部アセットなし」は
  rogue-13 で部分解除(モデルのみ)/「成長要素なし」は rogue-23 のスキル体系(永続マスタリー)で解除済み
