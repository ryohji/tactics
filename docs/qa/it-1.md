### ユーザに確認してほしい観点(統合担当の申し送り)
1. 初期表示で 地形ボリューム・占有セルの胞塗り・空きセル小球・active 金球 が噛み合っているか。
2. クリックで隣接到達セル(寒色)へ active が移り、再評価(reachable/胞)が走るか。到達外クリックが無反応か。
3. ドラッグで active 中心に旋回、ホイールで寄り引き([4S,19S])。ドラッグ直後のクリックで誤移動しないか。
4. leva の `d` を動かすと occluder(球の消失/胞塗り)が連続的に増減するか。`S`/`Hmax`/`Lmin`/`Lmax` も。
5. 「3D地形のヘックス分割・移動・表示が気持ちよくハマっているか」(VISION 北極星)。← it-1 の本質、人間の目で判断。

## 既知の制約・フォロー候補(欠陥ではない)
- 地形表示トグル(メッシュ/胞塗り/不透明度/res)が store 非管理で leva ローカル(W6)。it-1 は支障なし。将来 Scene で統括するなら `store.toggles` へ昇格。
- マーカー InstancedMesh の `onClick` は R3F が pointermove ごとに全インスタンスをレイキャスト(W5/W7 申し送り)。it-1 規模(数千)は問題なし。大規模化時は click 時のみのレイキャストへ要置換。
- FP視点・隣接稜線(菱形十二面体エッジ)は枠のみ/リングのみ(○任意、it-1 必須外)。
