# FCC Rogue — 蟻巣迷宮

**▶ プレイ: https://ryohji.github.io/tactics/**

3D 空間を FCC 格子(面心立方格子)でボロノイ分割し、2次元ヘックスを3次元へ拡張した
ターン制ローグライクの試作。菱形十二面体のセルが空間を隙間なく充填し、各セルは
12近傍を持つ ——「水平はヘックス、高さは層で読む」直観のまま、蟻の巣状に
下へ下へ広がるダンジョンを探索する。

## 遊びかた

- 世界は「無限の土の塊」から発見済みの空洞だけがくり抜かれた洞窟。たいまつの明かりが
  届いた場所だけが開けていく。ダンジョンは進行に合わせて自動生成される
- プレイヤー1行動=1ターン。行動するたびに敵が動く(古典ローグ式)
- **層と関門**: 深度8ごとに1つの層。関門(層の境界)に近づくと崩落の警告が出て、
  境界を+2越えると上層が丸ごと崩落し**二度と戻れない**(セーブ・メモリを1層分に
  抑える技術的な根治でもある)。関門を越えるとスキルスロットが1つ増え、装着の
  組み替えができる「降りる前の儀式」になる
- 勝利条件はない。**どこまで深く潜れるか**(最深到達深度と討伐数がスコア)。層境界の
  警告帯に居る間だけ「⛏脱出」ができ、確定すると持ち物の遺物(巣の琥珀)を確実に
  持ち帰れる**生還**で終える(push-your-luck — 脱出せず死ぬと琥珀は失う)
- HP が尽きたら死亡。「再挑戦」で新しい巣が生成される
- **スキル**: マスタリー(武技/盾/拳闘/隠密/罠師/灯火/甲殻の7系統)は使うことで育つ
  永続メタで、**死んでも残る**。育ったマスタリーが解禁したノードだけが、ラン内の
  スキルスロット(初期2・関門ごとに+1・上限6)に装着できる。ラン開始直後の「支度」で
  自由に組み替え、関門で増えた新スロットは解禁済みノードから3択ドラフトを引く
- 死亡画面から結果(深度・死因・討伐・装備・シード)を X にポストできる(#蟻巣迷宮 #FCCRogue)
- タイトル/死亡画面でシードを入力すると同じ迷宮を再現できる(数字のほか合言葉も可。空欄=ランダム)
- 冒険は毎ターン自動保存される(localStorage)。タイトル画面の「続きから」で再開、
  「保存データを破棄」で消去。死亡・新規開始でも破棄される(手動セーブによる巻き戻しはなし)
- タイトル画面には「これまでの冒険」(ラン履歴の上位5件・行クリックで同シード再挑戦)・
  「📅本日の迷宮」(日付由来のシード)・「📖図鑑」(討伐/アイテム図鑑・実績)がある

### 操作

| 操作 | 効果 |
|---|---|
| 左ドラッグ / ホイール | 視点の回転 / 寄り引き |
| 青マーカーをクリック | 移動(2歩まで。1歩=1ターン) |
| バブルをクリック | ファストトラベル(自動歩行。敵に気づかれると中断) |
| 隣接する敵をクリック | 近接攻撃 |
| ホバー(マーカー/敵) | 同じ高さの移動範囲ヘックス / 敵の情報と距離 |
| TAB / Shift+TAB | 部屋内の敵へ視線を巡回(情報パネル付き)/ 逆順 |
| M / 🗺 | マップモード切替 |
| マップ中: ドラッグ / Space+ドラッグ / TAB(Shift で逆順) | 回転 / 移動 / 訪問済みの部屋を巡回(バブルで入り口へ移動) |
| 🔥 | 明かり3〜4段階(下記トレードオフ。「消す」は灯火スキル装着時のみ出現) |
| ✨ | 光の演出(ブルーム等)のオン/オフ(重い環境向け) |
| ⛏脱出 | 層境界の警告帯限定。確認の上ランを終え、遺物を確定して生還する |
| 罠クリック | スキル「遠隔回収」装着時、設置済みの自分の罠を回収する |
| 待機 | その場で1ターン過ごす |
| ❓ | 操作説明(ヘルプ画面) |
| 📖(タイトル画面) | 図鑑モーダル(討伐/アイテム/実績。未踏破は「???」) |

スマートフォンでも遊べる: 1本指ドラッグ=回転 / ピンチ=寄り引き / 2本指ドラッグ=移動。
マーカー・敵・バブルは **1度目のタップ=選択(情報表示)、2度目=実行** の2段階。
TAB の代わりに ◀▶ ボタンで敵・部屋を巡回する。

### ゲームシステム

- **明かりのトレードオフ**: 広げるほど視界と自然回復が増すが、敵に気づかれやすくなる。
  絞れば隠密寄りに。灯火スキルを装着すると4段階目「消す」(視界最小・回復なしだが
  ほぼ気づかれない)が選べる
- **敵の縄張り**: 敵は縄張りと固有の階層範囲を持ち、そこから出てまでは追ってこない。
  深く逃げると振り切れる。上下方向の気づきは特に敏感
- **アイテム品質と合成**: 同じアイテム・同じ品質の2つを合成すると品質 +1
  (+1 の強化には別の +1 が必要)。品質は攻/防/回復量/威力/持続に効く
- **深層とレア度**: 深く潜るほど強い種族が現れ、出土・ドロップも上位装備や
  高品質(+1〜+3)に寄っていく。深度24を超えると敵の hp/攻撃力が係数でスケールし
  続ける(「死は深さの必然」— プレイヤー側の上限(HP24固定・スロット6・障壁24)を
  最終的に必ず追い越す設計)
- **武器の個性**: 長槍は2歩先に届き、大鎚はリーチ内の敵をまとめて薙ぎ払う。
  防具枠には盾(回避+)も装備でき、盾と両手武器(長槍・大鎚)は排他
- **罠**: 棘(即ダメージ)・火炎(延焼)・幻惑(混乱)・恐慌(逃走)・眠り(昏睡)を
  足元か隣接セルに設置(橙マーカーで選択)。敵が踏むと発動
- **魔導砲塔 / 囮人形**: 自動砲撃する時限砲台と、敵のターゲットを吸う耐久制のデコイ。
  罠と組み合わせた「誘い込み」が強い
- **障壁**: HP とは別枠の上書き式シールド(8/16/24 の目安。張り直しは加算せず置き換え)。
  HP バー上のシアン表示。自然減衰はしないが層を跨ぐ(崩落)と消える
- **状態異常**: 敵は毒(HP を毎ターン削る。障壁を素通り)・混乱(移動が時々逸れる)を
  プレイヤーに、プレイヤーは罠経由で延焼・混乱・恐慌・昏睡を敵に付与できる。
  障壁の水薬(深度3〜)・解毒の水薬(深度4〜)が対抗手段として先に出土する
- **敵ロスター**: 通常種17種+層境界に35%で湧く門番3種(茸の王/巨人王/白き主。
  撃破でスキルスロット+1と高品質戦利品)。ネズミの群れ・毒ヘビ・障壁に倍打の酸粘体・
  不動で胞子を撒く胞子茸・射線判定つきの遠隔種(洞穴の術師)・重装甲の洞穴サソリ・
  背討ちや掏摸を無効化する「気配感知」持ち(鬼火・影)など、性質ごとの対抗手段は
  必ずその脅威より浅い深度に配置される
- **スキル(マスタリー×スロット)**: 武技/盾/拳闘/隠密/罠師/灯火/甲殻の7系統・
  全25ノード。マスタリーは使用実績で育つ永続資産(死んでも残る)で、育つほど
  選べるノードの候補が増える。実際に効果を発揮させるにはラン内のスキルスロットへ
  装着する必要があり、スロットは関門でしか組み替えられない
- **図鑑・実績・遺物**: タイトル画面の📖図鑑に討伐/アイテムの収集状況と実績8種が
  永続で貯まる。層の中間帯には遺物「巣の琥珀」が湧き、警告帯での⛏脱出で確定した
  ものだけが展示棚(タイトル画面)に加わる
- **探索支援**: 訪問済みの部屋は壁色が変わり、敵の居なくなった部屋は明るくなる。
  発見したアイテム・通路の入り口・確認済みの敵はバブル(名前+歩数)で示される

## 開発

```sh
npm install
npm run dev        # http://localhost:5173/
npx vitest run     # テスト(モデル・状態機械)
npx vite build     # 本番ビルド
npm run balance    # バランスシミュレータ(ヘッドレス統計。方策×シードの死亡分布・死因等)
```

main への push で GitHub Actions が自動ビルドし GitHub Pages へデプロイされる
(`.github/workflows/pages.yml`)。

### 技術

- Vite + TypeScript + React Three Fiber(three.js)+ zustand
- 幾何は `src/model/fcc.ts`(FCC 格子・12近傍・ワールド座標変換)が単一の土台
- ダンジョンは「掘削セル集合」(`src/model/dungeon.ts`)。空間を一辺12のスロットに分割し、
  広間(スロットに最大1つ)と通路(隣接スロット間のリンク)を **(シード, スロット座標)
  の純関数**として生成する。重なりは構造的に発生せず、同じシードなら探索順に依らず
  同じ迷宮になる(プレイ再現)。プレイヤーの接近は「無限に広がる決定済みの巣の
  どこまでを実体化するか」だけを決め、合流(ループ)も自然に生まれる
- 描画は「空洞の内表面」だけをメッシュ化し、視点手前をクリッピング平面で切って
  ステンシルキャップで断面を塗る(カットアウェイ)。詳細は `docs/DESIGN.md` §7
- バランスシミュレータ(`src/sim/`, `npm run balance`)は実ゲームと同じストアを
  fake timers+演出待ちゼロで数千ラン回すヘッドレス統計。ボット方策×シードで
  死亡深度分布・死因・種族別寄与を集計し、変更前後の A/B 差分で難度の変動を検出する
  (`docs/balance/` にベースライン)。詳細は `docs/DESIGN.md` §9
- 効果音・BGM(深度連動の生成アンビエント)は WebAudio 合成。
  [BGM 試聴室](https://ryohji.github.io/tactics/bgm.html)(`bgm.html`)で深度・シーン別の
  鳴りかたと、地域風スタイルの試作(ケルト/アラブ/エジプト/インド)を聴ける。岩肌・グロー等の
  テクスチャも canvas 生成(トライプレーナー投影)。SSAO+ブルーム+ビネットの
  ポストエフェクト付き(✨で切替)
- 外部アセットは 3D モデルのみ(`public/models/` に出典・ライセンスを同梱):
  プレイヤー = KayKit Adventurers の Knight(CC0, Kay Lousberg / kaylousberg.com)、
  敵 = Quaternius の CC0 モデル群(poly.pizza 経由)と
  「Ant」(Poly by Google, **CC-BY 3.0**)。詳細は `public/models/beasts/credits.json`

### リポジトリ構成

```
src/model/       幾何・ダンジョン生成・敵/アイテム定義(Three 非依存・テスト対象)
src/model/rogue/ ゲームのドメイン層(戦闘・敵AI・マスタリー/スキル・湧き。Three/React 非依存の純関数)
src/state/       zustand ストア(rogue.ts)・視点・アニメーション・マスタリー/図鑑/履歴の永続化
src/sim/         バランスシミュレータ(ヘッドレス統計。方策・ランナー・集計)
src/render/      R3F コンポーネント
src/ui/          DOM オーバーレイの HUD
docs/            VISION / REQUIREMENTS / DESIGN / ROADMAP / DECISIONS(経緯の記録)・balance(ベースライン)
```

`tactics` ブランチには前身の3Dタクティクス試作(FE式戦術戦闘)が残っている。
