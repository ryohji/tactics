# FCC Rogue — 蟻巣迷宮

**▶ プレイ: https://ryohji.github.io/tactics/**

3D 空間を FCC 格子(面心立方格子)でボロノイ分割し、2次元ヘックスを3次元へ拡張した
ターン制ローグライクの試作。菱形十二面体のセルが空間を隙間なく充填し、各セルは
12近傍を持つ ——「水平はヘックス、高さは層で読む」直観のまま、蟻の巣状に
下へ下へ広がるダンジョンを探索する。

## 遊びかた

- 世界は「無限の土の塊」から発見済みの空洞だけがくり抜かれた洞窟。たいまつの明かりが
  届いた場所だけが開けていく。ダンジョンは進行に合わせて自動生成される
- プレイヤー1行動=1ターン。行動するたびに敵が動く(古典ローグ式)
- 勝利条件はない。**どこまで深く潜れるか**(最深到達深度と討伐数がスコア)
- HP が尽きたら死亡。「再挑戦」で新しい巣が生成される
- 死亡画面から結果(深度・死因・討伐・装備)を X にポストできる(#蟻巣迷宮 #FCCRogue)

### 操作

| 操作 | 効果 |
|---|---|
| 左ドラッグ / ホイール | 視点の回転 / 寄り引き |
| 青マーカーをクリック | 移動(2歩まで。1歩=1ターン) |
| バブルをクリック | ファストトラベル(自動歩行。敵に気づかれると中断) |
| 隣接する敵をクリック | 近接攻撃 |
| ホバー(マーカー/敵) | 同じ高さの移動範囲ヘックス / 敵の情報と距離 |
| TAB | 部屋内の敵へ視線を巡回(情報パネル付き) |
| M / 🗺 | マップモード切替 |
| マップ中: ドラッグ / Space+ドラッグ / TAB | 回転 / 移動 / 訪問済みの部屋を巡回(バブルで入り口へ移動) |
| 🔥 | 明かり3段階(下記トレードオフ) |
| ✨ | 光の演出(ブルーム等)のオン/オフ(重い環境向け) |
| 待機 | その場で1ターン過ごす |

### ゲームシステム

- **明かりのトレードオフ**: 広げるほど視界と自然回復が増すが、敵に気づかれやすくなる。
  絞れば隠密寄りに
- **敵の縄張り**: 敵は縄張りと固有の階層範囲を持ち、そこから出てまでは追ってこない。
  深く逃げると振り切れる。上下方向の気づきは特に敏感
- **アイテム品質と合成**: 同じアイテム・同じ品質の2つを合成すると品質 +1
  (+1 の強化には別の +1 が必要)。品質は攻/防/回復量/威力/持続に効く
- **罠**: 棘(即ダメージ)・火炎(延焼)・幻惑(混乱)・恐慌(逃走)・眠り(昏睡)を
  足元に設置。敵が踏むと発動
- **魔導砲塔 / 囮人形**: 自動砲撃する時限砲台と、敵のターゲットを吸う耐久制のデコイ。
  罠と組み合わせた「誘い込み」が強い
- **探索支援**: 訪問済みの部屋は壁色が変わり、敵の居なくなった部屋は明るくなる。
  発見したアイテム・通路の入り口・確認済みの敵はバブル(名前+歩数)で示される

## 開発

```sh
npm install
npm run dev        # http://localhost:5173/
npx vitest run     # テスト(モデル・状態機械)
npx vite build     # 本番ビルド
```

main への push で GitHub Actions が自動ビルドし GitHub Pages へデプロイされる
(`.github/workflows/pages.yml`)。

### 技術

- Vite + TypeScript + React Three Fiber(three.js)+ zustand
- 幾何は `src/model/fcc.ts`(FCC 格子・12近傍・ワールド座標変換)が単一の土台
- ダンジョンは「掘削セル集合」(`src/model/dungeon.ts`)。広間+掘りかけ通路(スタブ)の
  グラフを、プレイヤーの接近に応じて seed 決定的に生成する
- 描画は「空洞の内表面」だけをメッシュ化し、視点手前をクリッピング平面で切って
  ステンシルキャップで断面を塗る(カットアウェイ)。詳細は `docs/DESIGN.md` §12
- 効果音・BGM(生成アンビエント)は WebAudio 合成、グロー等のテクスチャも canvas 生成
  —— 外部アセットなし。ブルーム+ビネットのポストエフェクト付き(✨で切替)

### リポジトリ構成

```
src/model/   幾何・ダンジョン生成・敵/アイテム定義(Three 非依存・テスト対象)
src/state/   ゲーム状態機械(zustand)・視点・アニメーション
src/render/  R3F コンポーネント(rogue/ 配下がローグライク用)
src/ui/      DOM オーバーレイの HUD
docs/        VISION / REQUIREMENTS / DESIGN / ROADMAP / DECISIONS(経緯の記録)
```

`tactics` ブランチには前身の3Dタクティクス試作(FE式戦術戦闘)が残っている。
